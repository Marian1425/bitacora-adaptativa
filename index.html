    Conceptos.

    DOM(Document Object Model.)

    Es una representación en forma de árbol de todo lo que contiene una pagina web:

    • Texto.
    • Etiquetas HTML.
    • Atributos.

    DOM Animado.

    Modificar las propiedades de los elementos (su posición, tamaño, color, opacidad, etc.) para que cambien con el tiempo y creen la ilusión de movimiento.

    DOM de Eventos.

    Se encarga de manejar cómo los elementos de una página web reciben e interactúan con eventos como clics, movimientos del mouse, teclas presionadas, etc.

    DOM CSS

    Parte del DOM que describe y controla los estilos de una página.


    ¿Para qué sirve "viewport"?

    Sirve para indicarle al navegador cómo debe controlar la escala y el diseño de una página web en diferentes dispositivos, especialmente en móviles, para asegurar que el contenido sea visible, legible y se adapte al ancho de cada pantalla.
    Es fundamental para el diseño responsivo, ya que le dice al navegador qué ancho debe tener la ventana gráfica del dispositivo y qué escala inicial debe mostrarse el documento, evitando así que el usuario tenga que hacer zoom.

    Unidades de medida estáticas y relativas.

    Absolutas.

    Son las unidadesd e medida en CSS que realizan alusión a las unidades que no cambian, aquellas que en todos los entornos son equivalentes.

    • in: hace referencia a las pulgadas.
    • cm: se refiere a los centímetros.
    • mm: hace referencia a los milimetros.
    • bq: se refiere a un cuarto de la unidad mm.
    • pt: un punto es igual a 1/72 de una pulgada.
    • pc: una pica es igual a 12 puntos.
    • px: esta etiqueta se refiere a los pixeles que, aunque son absolutos, también son relativos a la densidad de la pantalla.

    Relativas.

    Son un pco más flexibles, es decir, se ajusta de consenso al tamaño de otro costo de alusión.

    • em: tamaño de fuente establecida en navegador.
    • ex: mitad del tamaño de fuente del navegador.
    • ch: la anchura del caracter "0".
    • %: porcentaje.
    • rem: tamaño de feunte personaliazdo.


    Animaciones.

    • Color Animation. No es un término estándar, sino que se refiere al uso de la teoría del color y la técnica de "ciclo de color" para crear efectos visuales en la animación. La teoría del color permite a los animadores evocar emociones, definir personajes y construir mundos, mientras que el ciclo de color es una técnica específica donde una secuencia de cambios de color en píxeles inmóviles crea la ilusión de movimiento, como en un reflejo del agua o en el brillo de luces.
    • Delay. El "delay" o retardo se refiere a un tiempo de espera que se puede aplicar a diferentes elementos y acciones, como el inicio de una animación, el efecto de una transición o la ejecución de una función de JavaScript.
    • Animation-timing-fuction. Define la curva de velocidad o aceleración de una animación, controlando cómo cambia su ritmo a lo largo del tiempo, desde el estado inicial al final.
    • Animation-direction. Determina si una animación se reproduce hacia adelante, hacia atrás o alternando entre ambas direcciones en cada ciclo.
    • Animation-fill-mode. Determina el estado de un elemento antes y después de que se ejecute una animación, controlando si los estilos del primer o último fotograma clave (keyframes) se mantienen o si el elemento vuelve a su estilo original.


    Wireframes

    • Prototipo de baja fidelidad WIREFRAMES.Representación simple, generalmente en papel o bocetos digitales.
    • De alta fidelidad WIREFRAMES FINOS. Se asemeja mucho al producto final en diseño y funcionalidad.
    • ¿En qué consiste el prototipado?
      • Definición de requisitos.
      • Creación del prototipo.
      • Pruebas y validación.
      • Iteración y mejora.
    • Prototipos de baja fidelidad (LO-FI). Se enfocan en la estructura y la funcionalidad general sin detalles visuales avanzados ni interactividad compleja. Son ideales para validar ideas de manera rápida y económica antes de invertir en un diseño detallado.
    • Tipos de prototipos de baja fidelidad.
      • Bocetos a mano (Sketching). Se usa para la lluvia de ideas.


      • Wireframes estáticos. Representan la disposición de elementos como botones, menús y formularios.


      • Mapa del sitio. Secuencia de ilustraciones que muestran el flujo de interacción de un usuario.


    • Nota. Imágenes siempre van con una equis.


    Mobile First.

    • Prioriza el diseño y desarrollo para dispositivos móviles (smartphones, tablets) antes que para computadoras. Se empieza con la pantalla más pequeña y se va escalando.
    • Responde al creciente uso de internet desde móviles. En México el 90% de los usuarios de internet se conectan vía smartphone.
    • Mejor Experiencia de Usuario (UX): Sitios más rápidos, fáciles de usar y adaptados a pantallas táctiles.
    • SEO.
    • Mayor rendimiento: Al iniciar con menos elementos, se logra una carga más rápida.
    • Foco en el Contenido Esencial. Obliga a priorizar la información mas importante para pantallas pequeñas.



    Logo.

    • Logotipo. Solo texto.
    • Isotipo. Solo símbolo.
    • Imagotipo. Texto y símbolos combinados.
    • Isologo. Texto y símbolo como uno solo. No es recomendado en la web ya que es difícil la lectura.
    • Indicadores de un buen logo.
      • Se debe reproducir a una tinta.
      • Tiene que tener escalabilidad.
      • Utilizar una retícula para su diseño.
    • La “X” o cualquier otra marca que se use en la retícula sirve para indicar la distancia mínima que debe respetarse alrededor del logotipo.


    Wireframes Finos.

    • Prototipos de alta fidelidad / Wireframes Finos. Se refiere a la creación de un modelo o “maqueta” interactiva que se asemeja mucho al producto final. A diferencia de los prototipos de baja fidelidad (como los bocetos en papel), estos prototipos son visualmente detallados y funcionales.

    • Wireframes finos. Se utilizan los Mockups.

    Display Flex

  • Order.Cambia el orden visual de los elementos felx sin mover el HTML. Sirve para reorganizar elementos sin cambiar el código.
  • Flex-grow.Controla cuánto crece cada elemento para ocupar espacio diponible. Sirve para hacer que algunos elementos sean más grandes.
  • Flex-basis.Define el tamaño base (ancho o alto inicial) antes de aplicar Flex-grow. Sirve para establecer el tamaño inicial de cada caja en un contenedor flex.
  • Align-self.Permite alinear individualmente un solo elemento aunque los demás estén distintos. Sirve para mover una caja independientemente de las otras.
